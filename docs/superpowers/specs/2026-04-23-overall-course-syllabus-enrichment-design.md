**Date:** 2026-04-23
**Author:** danth-ai-swe
**Status:** Draft (awaiting user approval)

## 1. Background & Goal

The `overall_course` Qdrant collection currently holds 5 chunk types (`node`, `lesson`, `module`, `course`, `overview`) generated by `src/rag/ingest/build_overall_chunks.py` from `data/metadata_node.json`. The builder only knows about concept nodes вЂ” it has no visibility into course-level syllabus info (objectives, outcomes, passing criteria, total hours) or lesson-level learning objectives.

Two Syllabus Excel files are now available and cover that gap:

- `data/LOMA281_Syllabus_1.0.xlsx`
- `data/LOMA291_Syllabus_1.0.xlsx`

**Goal:** Enrich the existing `course`, `module`, and `lesson` chunks with syllabus data so that queries like "What are the learning objectives of LOMA 281 Module 2 Lesson 1?", "How many hours is LOMA 291?", or "What does Module 1 of LOMA 281 cover?" hit the right chunks. Keep `chunk_type` names and deterministic IDs stable so a re-ingest overwrites points in place.

## 2. Scope

In scope:

- New module `src/rag/ingest/load_syllabus.py` that parses both Excel files and emits a normalized intermediate file `data/ingest_overall/syllabus.json`.
- Changes to `src/rag/ingest/build_overall_chunks.py` to read `syllabus.json` (if present) and enrich existing `course`, `module`, `lesson` chunks вЂ” both text body and `payload`.
- Regenerate `data/ingest_overall/overall_course.json` and verify the run.
- New constants in `src/constants/app_constant.py`: paths for the Syllabus sources and the intermediate JSON.

Out of scope:

- Pushing points to Qdrant (H2 decision вЂ” user triggers the `/documents/ingest` endpoint manually).
- Changes to `node` and `overview` chunks (G1 decision).
- Sheets `Author and Rec of Changes` and `Reference` (no RAG value).
- Modifying `metadata_node.json` itself.
- Changes to any search/query path (`pipeline.py`, router, prompts).
- Unit tests (project has no unit test framework yet; verification is manual вЂ” run the build script and inspect the JSON output).

## 3. Architecture

Two independent Python modules, each with a single responsibility. `load_syllabus.py` is invoked from `build_overall_chunks.main()` at the start of each run вЂ” it unconditionally reparses both Excel files and rewrites `syllabus.json`, so a single `python -m src.rag.ingest.build_overall_chunks` invocation produces the final JSON. `load_syllabus.py` also exposes its own `main()` for standalone debugging.

### 3.1 `load_syllabus.py`

**Responsibility:** parse 2 Excel files в†’ produce a normalized dict-of-courses structure, keyed by LOMA course code, and write to `syllabus.json`.

**Discovery:** glob `data/LOMA*_Syllabus_*.xlsx`. Each file's course code is parsed from its filename: `LOMA281_Syllabus_1.0.xlsx` в†’ course code `LOMA 281` (insert space before digits). This code is the match key back to `metadata_node.json` (B1 decision).

**Output shape** (`syllabus.json`):

```json
{
  "LOMA 281": {
    "course_code": "LOMA 281",
    "topic_name": "Meeting Customer Needs with Insurance and Annuities - 3rd Edition",
    "version": "1.0",
    "effective_date": "2025-06-26",
    "training_method": "Self-study",
    "type_of_learners": "BA",
    "prerequisites": "",
    "objectives": ["Understand Risk and Insurance Concepts", "Insurance Regulation", ...],
    "outcomes": ["Understand Insurance Fundamentals", ...],
    "assessment": {
      "scheme": ["Quiz"],
      "passing_criteria": "80% quiz points"
    },
    "totals": {
      "self_learning_hours": 28,
      "quiz_hours": 7,
      "review_hours": 7,
      "total_hours": 42
    },
    "modules": {
      "1": {
        "module_num": 1,
        "title": "Risk and Insurance",
        "description": "In this module, we'll start with some basics. What is life insurance? ...",
        "lessons": {
          "1": {
            "lesson_num": 1,
            "title": "Risky Business?",
            "objectives": "Learning Objectives\nRisky Business\nTypes of Risk\n...",
            "self_learning_hours": 2.0,
            "quiz_hours": 0.5,
            "review_hours": 0.5,
            "delivery_mode": "Document self-learning",
            "directory": "Lesson 1",
            "remark": "Detail Lecture: LOMA281_M1L1_Risk and Insurance\nSummary: LOMA281_M1L1_Summary"
          }
        }
      }
    }
  },
  "LOMA 291": { ... }
}
```

**Parsing rules:**

- **Cover sheet:** scan rows for labels `Document Code`, `Version`, `Effective Date` in column F (index 5) and read the value in column I (index 8). Date cells в†’ ISO string `YYYY-MM-DD`.
- **Syllabus sheet:** rows are a key/value table. Column B (index 1) holds the item number, column C (index 2) holds the label (e.g., `Topic Name *`, `Course Objectives *`), column D (index 3) holds the main value, column E (index 4) holds sub-values (used for the `Expected Achievement` weights). Iterate rows, dispatch by label. Multi-line cells (`Course Objectives`, `Course Outcomes`) split on `\n` в†’ list of strings, whitespace-stripped, empty entries dropped.
- **Schedule sheet:** header row at index 1 (`Module | Lesson | Section/Objectives | Delivery Mode | Duration | PIC | Learning Format | Directory | Remark`). Forward-fill Module column (new module value appears only on the lesson's first `Document self-learning` row and stays blank for Quiz/Review rows below). Regex extracts `Module N:` + trailing text в†’ `module_num` + `title` + `description`. Same for `Lesson N:`. For each lesson, gather 3 consecutive rows: `Document self-learning` в†’ duration в†’ `self_learning_hours`; `Quiz` row в†’ `quiz_hours`; `Review` row в†’ `review_hours`. Directory + Remark read from the `Document self-learning` row. Last rows (`Document self-learning | 28`, `Quiz | 7`, `Review | 7`, `Total hours | 42`) are summary rows вЂ” detect by the label in column B, populate `course.totals`.
- **Vietnamese annotations in objectives (C1):** kept verbatim вЂ” no stripping.

### 3.2 `build_overall_chunks.py` enrichment

**Responsibility:** unchanged purpose (generate all chunks), but now enriches `course`, `module`, `lesson` chunks with fields from `syllabus.json` when a match exists.

**Course matching (B1):** for each chunk, extract the substring `LOMA \d+` from `course` field (existing chunk). Lookup `syllabus[code]`. If missing, enrichment is skipped for that chunk (graceful degradation вЂ” LOMA 281 + LOMA 291 should always match; other LOMA courses without syllabus files keep current behavior).

**Module/Lesson matching:** by `(course_code, module_num, lesson_num)`. The existing `_module_number()` / `_lesson_number()` helpers already extract the integers. `syllabus.json` modules/lessons are keyed by the same integers (as strings, since JSON keys must be strings).

**Chunk text changes:**

*`lesson` chunk (D1)* вЂ” append a `Learning Objectives (from syllabus):` block and a `Duration:` line after the existing `Lesson Summary (from source):` line. Payload gains `delivery_mode`, `directory`, `remark`, `self_learning_hours`, `quiz_hours`, `review_hours`.

```
Lesson 1 - Risky Business (Course: LOMA 281 ..., Module 1 - Risk and Insurance)
Number of nodes: 15
Concepts covered:
- Risk: ...
- Speculative Risk: ...

Lesson Summary (from source): ...

Learning Objectives (from syllabus):
Learning Objectives
Risky Business

Risk Management

Duration: 2h self-study, 0.5h quiz, 0.5h review
```

*`module` chunk (E1)* вЂ” prepend a `Module Overview (from syllabus):` block immediately after the first header line (the `{module} (Course: ...)` line), before `Number of lessons:`.


*`course` chunk (F1)* вЂ” append 3 blocks after the existing `Main topics:` line.

Course payload gains: `version`, `effective_date`, `topic_code`, `type_of_learners`, `training_method`, `assessment_scheme`, `total_hours`.

### 3.3 Deterministic IDs

Existing UUID v5 scheme (`_det_id("course", course)`, `_det_id("module", course, module)`, `_det_id("lesson", course, module, lesson)`) is **unchanged**. Re-ingest with `force_restart=false` overwrites the same points; with `force_restart=true` drops and recreates cleanly. No orphaned points.

## 4. Data & Edge Cases

- **Missing syllabus for a course:** if `metadata_node.json` contains a course whose `LOMA \d+` code has no matching file, enrichment skipped silently вЂ” chunk produced exactly as today. Logged via `print()` (project convention, no logger in this script).
- **Mismatched module/lesson numbers:** if a module/lesson exists in `metadata_node.json` but not in Syllabus (e.g., new content not yet in syllabus), enrichment is skipped for that item; the chunk is still produced. Same in reverse: a Syllabus-only lesson without concept nodes would never surface as a chunk (no opportunity to enrich) вЂ” acceptable since Syllabus never adds lessons that aren't already taught.
- **Empty objectives cell:** lesson produced without the `Learning Objectives` block (don't print an empty section). Same for missing Module Overview.
- **`Selft-study` typo in source:** echoed verbatim into payload (not our job to correct source data; if we need to normalize later, that's a separate change).
- **Effective date cell as pandas Timestamp:** cast via `.strftime("%Y-%m-%d")`; if `NaT`, store empty string.
- **Duration cells with `NaN`:** coerce to `0.0`. All hour fields stored as `float` in `syllabus.json` for consistency (integer totals like `28` become `28.0`).
- **Duration line formatting in lesson chunk:** printed in `Xh` form, dropping trailing `.0` for whole numbers (e.g., `2h self-study, 0.5h quiz, 0.5h review`). Only printed when the lesson has any syllabus match; otherwise omitted entirely.
- **Vietnamese text encoding:** parse with default pandas `openpyxl` engine (supports UTF-8); write JSON with `ensure_ascii=False` (already the case).
